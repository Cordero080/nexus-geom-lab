## üìù Documentation Standards

### Naming Convention

- Use UPPERCASE_WITH_UNDERSCORES.md for guides
- Use kebab-case.md for code-specific docs
- Include emojis in headers for visual scanning

### Structure

- Start with overview/purpose
- Include table of contents for long docs
- Use code examples liberally
- End with key takeaways

### Markdown Best Practices

- Use headers hierarchically (don't skip levels)
- Code blocks should specify language
- Use tables for comparisons
- Use lists for steps/checklists
